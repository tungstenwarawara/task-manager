# Journal / 日記

今日の日記を書きます。

## トリガー

- `日記を書きたい`
- `日記`
- `今日の日記`
- `journal`
- `今日の振り返り`

## 使い方

対話形式で日記を記入し、knowledge-base リポジトリに保存します。

## 手順

1. 今日の日付を確認
2. ユーザーに以下の質問をする:
   - 今日やったことは何ですか？
   - 学んだことや気づきはありますか？
   - 明日やりたいことは？
   - 今日の気分は？（1-5）
   - 今日感謝したいことは？

3. 日記ファイルを作成:
   - パス: `knowledge-base/journal/YYYY/MM/DD.md`

4. Git でコミット・プッシュ:
```bash
cd knowledge-base
git add journal/
git commit -m "Add journal for YYYY-MM-DD"
git push
```

## テンプレート

```markdown
# YYYY-MM-DD (曜日)

## 今日やったこと
-

## 学んだこと
-

## 明日やること
-

## 気づき・感想


## 気分 (1-5):

## 今日の感謝
-

---
*Created: YYYY-MM-DD HH:MM*
```

## 例

```
> 日記を書きたい

Claude: 今日の日記を始めましょう。今日やったことを教えてください。

> プロジェクトの企画書を完成させた

Claude: 素晴らしいですね！何か学んだことや気づきはありましたか？

> 締め切りを意識すると集中力が上がることに気づいた

Claude: いい気づきですね。明日やりたいことはありますか？

> 企画書のレビューを受ける

Claude: 今日の気分を1-5で表すとどのくらいですか？

> 4

Claude: 日記を作成しました。
```
